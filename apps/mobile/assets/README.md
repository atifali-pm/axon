# Mobile Assets

The four PNGs in this folder are **placeholders** generated by PIL (see the recipe below). They're valid assets so `eas build` succeeds, but for store submission you'll want a real designed logo.

## Current placeholders

| File | Dimensions | Notes |
|---|---|---|
| `icon.png` | 1024×1024 | Concentric squares, blue accent on `#0a0a0a` |
| `adaptive-icon.png` | 1024×1024 | Same glyph with transparent background for Android adaptive icon |
| `splash.png` | 1242×2436 | Logo centred at 40% of width on dark background |
| `favicon.png` | 48×48 | Miniaturised icon for web build |

cd apps/mobile/assets
python3 - <<'PY'
from PIL import Image, ImageDraw
# … see the script in the repo history (git log -p -- apps/mobile/assets/README.md)

## Replacing with real artwork

1. Export your logo as a 1024×1024 PNG on a `#0a0a0a` background.
2. Overwrite `icon.png`.
3. For `adaptive-icon.png`, keep the inner content in the centre **672×672** safe zone so Android's circular, squircle, and rounded-square masks don't crop it.
4. For `splash.png`, place the logo at roughly the centre on a solid `#0a0a0a` canvas (1242×2436 covers the iPhone X-class; Expo scales for other devices).
5. `favicon.png` (48×48) is just for the `expo start --web` preview.

No code changes needed after replacing the files; `app.json` already references them by path.
